02 — Pipeline

End-to-End
Training Pipeline

5 stages from raw data to generated music. Small configs enable single-GPU prototyping; full configs scale to 8×A100.

Stage 01
Data Preparation
Downloads POP909 (909 files), Lakh MIDI (~17k), MAESTRO v3 (~200h piano). Tokenises with REMI+ 512-token vocabulary. Renders MIDI to WAV via FluidSynth. Generates synthetic preference pairs.
bash 01_prepare_data.sh
Stage 02
Symbolic Model Training
Trains the Hierarchical Music Transformer from random initialisation. Small config: ~15M params, 1–3h single GPU. Full config: ~80M params, 48–72h 8×A100.
python 02_train_symbolic.py --config config/symbolic_small.yaml
# Full: torchrun --nproc_per_node=8 02_train_symbolic.py --config config/symbolic_full.yaml
Stage 03
Audio Renderer Training
Trains the Mel-Spectrogram Diffusion U-Net conditioned on symbolic tokens. Requires FluidSynth-rendered audio pairs from Stage 1. ~25M parameters.
python 03_train_renderer.py --config config/renderer_small.yaml
Stage 04
DPO Alignment
Fine-tunes the symbolic model with Direct Preference Optimisation using heuristic-scored synthetic preference pairs (or human-labelled data if available). Beta = 0.1 KL penalty.
python 04_preference_alignment.py \
    --symbolic_ckpt checkpoints/symbolic/latest.pt \
    --pref_data   data/subsets/preference_data.json
Stage 05
Song Generation
Generates MIDI + WAV using text prompt, style, emotion, and tempo. Supports nucleus sampling, temperature, and optional neural render bypass.
python 05_generate_song.py \
    --prompt    "epic orchestral battle theme with brass and percussion" \
    --style     CLASSICAL --emotion TENSE --tempo 140
# Styles: POP | CLASSICAL | JAZZ | FOLK | ELECTRONIC | AMBIENT
# Emotions: HAPPY | SAD | TENSE | PEACEFUL

06 — Transparency

Technical Depth
& Limitations

📋 Prototype Disclosure

Note: The provided outputs represent sample results generated using a prototype implementation. Due to computational constraints, full-scale training was not performed, and the outputs are included to demonstrate the system pipeline and generation capability.

🖥️
Compute Requirements
Full-scale training requires 8× NVIDIA A100/A800 (80GB VRAM), ~500GB storage, and 48–96h training time. Prototype uses single-GPU small configs.
🎵
Output Quality
Small model outputs demonstrate pipeline functionality, not commercial quality. Audio rendered via Griffin-Lim vocoder. Neural render bypass available via --skip_neural_render.
🔒
License
This project is now released under a proprietary license. All rights reserved. See the LICENSE file in the repository for details.
🔧
Common Issues
FluidSynth required for MIDI-to-audio. CUDA OOM → reduce batch size in YAML. Poor quality → train more steps or use larger dataset split.
07 — Quick Start

Get
Started

Two commands from clone to generated music. First run takes 5–10 minutes for environment setup.

Step 1 — Setup
# Run setup (installs everything — ~5-10 minutes first time)
bash setup.sh
Step 2 — Generate
# Run full pipeline (data + train + generate)
bash run_full_pipeline.sh
Outputs Location
# After completion, find your generated music in outputs/:
outputs/
  song_01_pop_happy.mid / .wav
  song_02_classical_sad.mid / .wav
  song_03_ambient_peaceful.mid / .wav
  song_04_jazz_tense.mid / .wav
Custom Generation
conda activate ai-music-gen
python 05_generate_song.py \
    --prompt   "epic orchestral battle theme with brass and percussion" \
    --style    CLASSICAL \
    --emotion  TENSE \
    --tempo    140 \
    --bars_per_section 16 \
    --output_name my_custom_song
# --temperature 0.8 # lower = more conservative
# --top_p 0.85 # nucleus sampling p
# --skip_neural_render # force FluidSynth output
Monitor Training
# TensorBoard
tensorboard --logdir runs/
# Check checkpoints
ls -lh checkpoints/symbolic/
ls -lh checkpoints/renderer/
ls -lh checkpoints/aligned/
